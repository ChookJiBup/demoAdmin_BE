# Spring Backend 아키텍처 및 구조 가이드

## 1. 목적

이 문서는 Spring 기반 백엔드 프로젝트를 새로 만들거나 확장할 때
개발자와 Coding Agent가 동일한 파일 구조와 코드 작성 방식을 따르도록 한다.

특정 서비스명, 프로젝트명, 패키지명에 의존하지 않으며 다음 유형을 대상으로 한다.

- Java와 Spring Boot 기반
- JPA 기반 영속성 처리
- 도메인 중심 패키지 구조
- Command와 Query 책임 분리
- 단위·통합·동시성 테스트 운영
- Swagger/OpenAPI 기반 API 문서 제공
- Redis, 메시지 브로커, 검색 엔진 등 외부 인프라 연동
- 향후 모듈 분리 또는 MSA 전환 가능성 고려

핵심 목표는 다음과 같다.

1. 기능이 늘어나도 파일 위치를 예측 가능하게 유지
2. 도메인 규칙과 프레임워크 코드를 분리
3. 쓰기와 읽기의 변경 이유를 분리
4. 계층별 테스트 범위를 명확하게 구분
5. AI가 기존 구조를 깨지 않고 코드를 추가하도록 유도

# 2. 핵심 아키텍처

## 2.1 도메인 우선 패키지 구조

최상위 패키지를 기술 계층으로 나누지 않는다.

나쁜 예:

```text
controller/
service/
repository/
entity/
dto/
```

권장 예:

```text
api/
admin/
domain-a/
domain-b/
domain-c/
auth/
common/
global/
```

HTTP 진입점은 최상위 `api/` 패키지에 도메인별로 모은다.
각 도메인 안에서는 application, domain, infrastructure,
support 계층을 중심으로 나눈다.

```text
api/
└── domain-a/
    ├── DomainAController.java
    └── dto/

domain-a/
├── command/
├── query/
└── support/
```

도메인 위치 판단 기준:

- 계정, 역할, 소속, 상태처럼 관리 대상 자체의 규칙은 해당 도메인에 둔다.
- 로그인, 토큰 발급, 인증 필터처럼 인증 행위와 보안 흐름은 `auth/`에 둔다.
- 예를 들어 관리자 계정 Aggregate와 관리자 역할은 `admin/` 도메인에 두고, JWT 로그인 유스케이스는 `auth/`에 둔다.
- 행사 진행자, 현장 운영자, 아르바이트생처럼 운영 코드를 받아 제한 권한으로 접속하는 사용자는 관리자 계정에 포함하지 않고 추후 `operator/` 도메인으로 분리한다.

장점:

- 기능 변경 범위를 한 도메인 안에서 추적 가능
- 도메인 간 내부 구현 직접 참조 감소
- 코드 소유권과 리뷰 범위 명확화
- 모듈 분리 및 MSA 전환 용이
- AI가 관련 파일을 빠르게 탐색 가능

## 2.2 Command와 Query 분리

### Command

상태를 바꾸는 작업을 담당한다.

- 생성
- 수정
- 삭제
- 상태 전이
- 트랜잭션
- 도메인 규칙 검증
- 동시성 제어
- 이벤트 발행

### Query

데이터를 읽는 작업을 담당한다.

- 단건 조회
- 목록 조회
- 검색
- 필터링
- 정렬
- 페이징
- 조회 DTO 조립
- Querydsl 또는 전용 SQL

권장 구조:

```text
sample/
├── command/
│   ├── application/
│   ├── domain/
│   └── infrastructure/
└── query/
    ├── application/
    ├── repository/
    └── infrastructure/
```

완전한 CQRS를 구현하지 않더라도,
쓰기 모델과 읽기 모델의 책임은 코드 수준에서 구분한다.

---

## 2.3 의존 방향

```text
api → application → domain
infrastructure → domain/application port
```

금지:

```text
domain → controller
domain → JpaRepository
domain → RedisTemplate
domain → WebClient
domain → 메시지 브로커 SDK
```

도메인은 HTTP, DB, Redis, Kafka 같은 기술을 몰라야 한다.
