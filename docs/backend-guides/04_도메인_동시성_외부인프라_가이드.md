# Spring Backend 도메인 동시성 외부 인프라 가이드

# 7. 상태 전이

```java
public enum ResourceStatus {

    CREATED,
    ACTIVE,
    PROCESSING,
    COMPLETED,
    CANCELLED;

    public boolean canActivate() {
        return this == CREATED;
    }

    public boolean canCancel() {
        return this == CREATED || this == ACTIVE;
    }

    public boolean isTerminal() {
        return this == COMPLETED || this == CANCELLED;
    }
}
```

테스트 대상:

- 정상 전이
- 순서 건너뛰기
- 완료 후 재변경
- 취소 후 재활성화
- 동일 요청 반복
- 동시 요청
- 권한 없는 요청

상태 전이는 Service 테스트뿐 아니라 Aggregate 단위 테스트로도 검증한다.

# 8. 동시성 제어

먼저 보호할 불변식을 정의한다.

예:

- 하나의 작업에는 한 명만 배정
- 수량은 음수가 될 수 없음
- 동일 요청은 한 번만 승인
- 취소와 완료가 동시에 성공할 수 없음
- 재고보다 많이 차감할 수 없음

## 비관적 락

충돌이 잦고 트랜잭션이 짧을 때 고려한다.

```java
@Lock(LockModeType.PESSIMISTIC_WRITE)
@Query("""
    select r
    from Resource r
    where r.id = :id
""")
Optional<Resource> findByIdForUpdate(@Param("id") Long id);
```

## 낙관적 락

충돌이 드물고 재시도 가능한 경우 고려한다.

```java
@Version
private Long version;
```

## 분산 락

적용 전에 다음을 먼저 검토한다.

- DB 원자 연산
- unique constraint
- 조건부 update
- 멱등키
- 메시지 파티션 키

분산 락을 첫 번째 선택으로 사용하지 않는다.

# 9. 외부 인프라

## Redis

```text
infrastructure/redis/
├── RedisConfig.java
├── ResourceCacheRepository.java
├── ResourceRedisRepository.java
└── ResourceRedisEntity.java
```

규칙:

- Service에서 Redis 키 문자열 직접 조립 금지
- TTL 명시
- 캐시 miss 정책 정의
- DB와 캐시 정합성 전략 정의
- Redis 장애 시 fallback 정책 정의

## 메시지 브로커

```text
infrastructure/messaging/
├── producer/
├── consumer/
├── message/
└── config/
```

```java
public interface ResourceEventPublisher {
    void publish(ResourceCreatedEvent event);
}
```

규칙:

- 도메인 이벤트와 전송 메시지 DTO 분리
- Consumer는 Application만 호출
- Consumer에 핵심 규칙 구현 금지
- 중복 소비를 가정한 멱등성 확보
- retry와 DLQ 정책 분리
- 필요 시 Outbox 패턴 사용

## 외부 API

```text
infrastructure/client/
├── ExternalClient.java
├── ExternalClientAdapter.java
├── ExternalRequest.java
├── ExternalResponse.java
└── ExternalExceptionTranslator.java
```

규칙:

- 외부 응답 모델을 내부 도메인에 직접 사용 금지
- timeout, retry, circuit breaker 명시
- 재시도 가능 오류와 불가능 오류 구분
- 외부 오류를 내부 ErrorCode로 변환
- 로그에 토큰과 개인정보 출력 금지
